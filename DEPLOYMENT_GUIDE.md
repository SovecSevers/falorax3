## 🚀 Шаги для полного развертывания

### Шаг 1: Задеплойте Edge Function

**Это критически важно!** Без этого шага база данных не будет работать.

1. Откройте **Настройки Make** (Settings) в Figma Make
2. Найдите раздел **Supabase**
3. Нажмите кнопку **"Deploy Edge Function"** или **"Redeploy Edge Function"**
4. Дождитесь завершения деплоя (10-30 секунд)
5. Вы должны увидеть сообщение об успешном деплое

**Почему это важно:**
- Edge Function — это серверная часть, которая обрабатывает все запросы к базе данных
- Без деплоя фронтенд не может подключиться к Supabase
- После каждого изменения `/supabase/functions/server/index.tsx` нужно передеплоить

---

### Шаг 2: Получите публичную ссылку

У вас есть **два варианта** для публичного доступа:

#### Вариант A: Использовать URL Figma Make (Рекомендуется для быстрого старта)

1. Откройте ваш проект в **Figma Make**
2. В правом верхнем углу найдите кнопку **"Share"** или **"Preview"**
3. Скопируйте публичную ссылку вида:
   ```
   https://make.figma.com/app/XXXXXXX
   ```
4. Эта ссылка работает сразу и доступна всем

**Плюсы:**
- Работает сразу
- Не нужно настраивать хостинг
- Автоматические обновления при изменениях

**Минусы:**
- Зависит от Figma Make
- URL не кастомный

#### Вариант B: Развернуть на собственном домене (Для продвинутых)

Если хотите свой домен типа `vanillix.yourdomain.com`, можно экспортировать код и развернуть на:
- **Vercel** (рекомендуется)
- **Netlify**
- **Cloudflare Pages**

Инструкции доступны по запросу.

## 🔐 Система прав доступа

### Роли пользователей

1. **Админ (вы)**
   - Логин: `admin`
   - Пароль: `admin123` (можно изменить в коде)
   - Может: создавать, редактировать, удалять кланы/города
   - Может: управлять правами доступа других игроков

2. **Редакторы** (доверенные игроки)
   - Логин: их игровой ник
   - Пароль: не требуется (если вы добавили их в белый список)
   - Может: создавать, редактировать, удалять кланы/города
   - Не может: управлять правами доступа

3. **Обычные пользователи** (все остальные)
   - Не нужно входить
   - Может: только просматривать контент
   - Не может: ничего менять

### Как добавить редактора

1. Войдите как **admin** (логин: `admin`, пароль: `admin123`)
2. Нажмите кнопку **"Права доступа"** в правом верхнем углу
3. Введите игровой ник игрока в поле "Введите ник игрока"
4. Нажмите **"Добавить"**
5. Теперь этот игрок может войти, используя свой ник (пароль не нужен)

### Как изменить админ пароль

1. Откройте файл `/supabase/functions/server/index.tsx`
2. Найдите строку:
   ```typescript
   adminPassword: "admin123"
   ```
3. Измените `admin123` на свой пароль
4. **Важно:** Задеплойте Edge Function заново (Шаг 1 выше)

## 🔄 Как работает синхронизация

### База данных
- Все данные хранятся в **Supabase** (облачная база данных)
- База работает **24/7** независимо от вас
- Изменения видны **всем пользователям мгновенно**

### Что сохраняется
- ✅ Кланы (название, глава, участники, координаты, фото и т.д.)
- ✅ Города (название, мэр, жители, координаты, фото и т.д.)
- ✅ Права доступа (список редакторов)

### Что НЕ сохраняется
- ❌ Статус входа пользователя (хранится локально в браузере)
- ❌ Временные данные форм (очищаются после сохранения)

## 🛠️ Устранение неполадок

### Проблема: "Error loading data: TypeError: Failed to fetch"

**Решение:**
1. Убедитесь, что Edge Function задеплоена (см. Шаг 1)
2. Проверьте консоль браузера (F12) на ошибки
3. Попробуйте передеплоить Edge Function

### Проблема: "Изменения не сохраняются"

**Решение:**
1. Убедитесь, что вы вошли в систему
2. Проверьте, что вы в списке редакторов
3. Проверьте консоль на ошибки API

### Проблема: "Игроки не видят мои изменения"

**Решение:**
1. Попросите игроков обновить страницу (F5)
2. Проверьте, что изменения сохранились (нет ошибок в консоли)
3. Убедитесь, что все используют одну и ту же ссылку

### Проблема: "Забыл админ пароль"

**Решение:**
1. Откройте Supabase Dashboard: https://supabase.com/dashboard
2. Найдите ваш проект
3. Перейдите в **Database** → **Tables** → `kv_store_96bf23b4`
4. Найдите запись с `key = "permissions"`
5. Измените `adminPassword` в JSON значении
6. Или просто измените в коде и передеплойте (см. "Как изменить админ пароль")
